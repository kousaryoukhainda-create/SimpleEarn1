HOW TO BUILD APK:

Option 1: Using Android Studio (Recommended)
─────────────────────────────────────────────
1. Extract SimpleEarn.zip
2. Open folder in Android Studio
3. Wait for Gradle sync
4. Click ▶ Run button
5. APK builds automatically

Output: app/build/outputs/apk/debug/app-debug.apk


Option 2: Using Command Line
─────────────────────────────
Requirements:
  - Android SDK installed
  - ANDROID_SDK_ROOT or ANDROID_HOME environment variable set
  - Java 17+ (already included)

Steps:
1. Extract SimpleEarn.zip
2. Navigate to project folder
3. Run: ./BUILD_APK.sh
4. Or directly: ./gradlew assembleDebug

Output: app/build/outputs/apk/debug/app-debug.apk


Option 3: Using Gradle Directly
────────────────────────────────
./gradlew assembleDebug          # Debug APK (~15 MB)
./gradlew assembleRelease        # Release APK (~13 MB, optimized)
./gradlew clean                  # Clean build
./gradlew installDebug           # Install on emulator

ENVIRONMENT SETUP:
═══════════════════════════════════════════════════════════════════════════

Required for building:
  - Android SDK (API 34)
  - Android NDK (optional)
  - Java 17 or higher

Set environment variables (Linux/Mac):
  export ANDROID_SDK_ROOT=/path/to/Android/Sdk
  export PATH=$PATH:$ANDROID_SDK_ROOT/platform-tools

Set environment variables (Windows):
  set ANDROID_SDK_ROOT=C:\Users\YourName\AppData\Local\Android\Sdk
  set PATH=%PATH%;%ANDROID_SDK_ROOT%\platform-tools


TROUBLESHOOTING:
════════════════════════════════════════════════════════════════════════════

Problem: "Android SDK not found"
Solution:
  export ANDROID_SDK_ROOT=/path/to/Android/Sdk
  ./gradlew assembleDebug

Problem: "gradle: command not found"
Solution:
  Use ./gradlew instead of gradle
  ./gradlew assembleDebug

Problem: "Gradle sync failed in Android Studio"
Solution:
  1. File → Invalidate Caches → Invalidate and Restart
  2. Wait for restart
  3. File → Sync Now

Problem: "Out of memory during build"
Solution:
  export GRADLE_OPTS="-Xmx4096m"
  ./gradlew assembleDebug

Problem: "Build fails with 'Unsupported Java version'"
Solution:
  Ensure you have Java 17+
  java -version
